## 1 · Typography / design system (applies globally)

DM Serif Display has been **deleted from the deck entirely**. The whole document is now single-family **Work Sans** with brand-spec weights and tracking.

| Role | Font | Weight | Tracking |
|---|---|---|---|
| Heading (H1, cover title, division title, stat values, `.serif` display utility) | Work Sans | **700** | **−0.02em** |
| Sub-heading (`.lede`) | Work Sans | **300** | **−0.04em** |
| Body copy | Work Sans | **300** | **0** |
| Label microcopy (`.pageheader-eyebrow`, chips) | Work Sans | 800 | (unchanged) |
| Mono (data values) | inherited mono stack | 800 | (unchanged) |

**Google Fonts link** now loads only Work Sans 300 / 400 / 500 / 600 / 700 / 800 / 900. Drop any DM Serif Display references in the production pipeline.

**Tracking rule:** use em-based letter-spacing (`-0.02em` / `-0.04em`) for display + sub-heading types so the tracking scales with size. Do not fix it in px.

**Inline display blocks** (originally `font-family:'DM Serif Display'`, now `'Work Sans'`) had no explicit `font-weight`. They were auto-promoted to `font-weight:700` so they don't inherit body's new 300 Light and render anaemic at 30pt+. Any new display text added inline must carry an explicit `font-weight:700`.

## 4 · Build pipeline notes (unchanged but worth restating)

1. **Source:** `hs-board-pack-mockup/index.html` (single file, inline CSS in `<style>`).
2. **Render:** Cloudflare Browser Rendering invokes Chromium headless against the deployed Pages URL with `print-to-pdf` + page-size override matching `@page { size: 338.67mm 190.5mm }`.
3. **Output:** PDF uploaded to R2 (`r2://hs-board-pack/{engagementId}.pdf`). Signed URL returned to Riley.
4. **Local dev parity:**
   ```bash
   chrome --headless=new --no-pdf-header-footer --no-margins \
          --print-to-pdf=out.pdf file:///path/to/index.html
   ```
5. **Renumber pass:** any add / move / hide of a `<section class="page">` requires a renumber pass to refresh every `.footer-pg` cell to `NN of {total}`. (Visible section count drives the total; `display:none` sections are skipped.)
